Step-by-Step Guide

Learn how to easily configure and run manual or completely automated AI bias evaluations.

1. Initial Setup

1

Navigate to the Testing Hub: From the homepage, click the "Start Testing Models" button to seamlessly navigate to the Configuration Page.

2

Enter Model Details: Type in the precise name of the AI model you wish to evaluate (e.g., GPT-4, Llama-3, Claude Opus).

3

Select Evaluation Scope: Choose the volume of questions per category. We recommend 5-20 questions for manual testing and 50+ questions for automated deep-dives. Then, verify the bias categories you'd like to test (e.g., Gender or Age Bias).

4

Choose Workflow Strategy: Finalize by selecting either "Manual Test" (you provide responses yourself) or "Automate API" (Trust AI runs it for you). See below for detailed flows.

2. Manual Testing Workflow

Use this mode if you do not have API access to the model, or if you prefer to copy/paste queries directly into chat interfaces like ChatGPT or Claude.

1

Click "Manual Test": After your initial setup, clicking this executes the system to generate distinct, targeted prompts mapping to your selected bias parameters.

2

Copy the Prompts: You will be presented with a list of generated statements. Copy the first prompt.

3

Query Your External AI: Switch to your AI model's chat screen (e.g., ChatGPT browser). Paste the copied prompt and wait for the AI to provide its completion.

4

Paste & Continue: Copy the text response from your AI and paste it back into the Trust AI response box. Once all boxes have been answered, click "Get Results" to analyze biasness, consistency, and fairness.

3. Automation Testing Workflow

Use this mode to directly link an AI Provider API endpoint. The platform automatically sends all queries concurrently, saving hours of manual labor.

1

Click "Automate API": Navigate to the Integration page from the initial setup form.

Understanding the Automation Form Fields

A

API URL: Specify the exact request endpoint for your LLM or custom model provider. (Obtain this from your provider docs)

Example: https://api.openai.com/v1/chat/completions
B

HTTP Method: Select from GET or POST. (Language model generation almost entirely relies on POST requests).

C

Headers: These are the HTTP configuration parameters. Most importantly, you place your Authorization API keys here. Ensure valid JSON mapping.

{ "Content-Type": "application/json", "Authorization": "Bearer YOUR_SECRET_API_KEY" }
D

Request Body Template: The exact JSON shape describing how parameters are sent to the AI. Trust AI specifically looks for the placeholder {{question}}. This is incredibly important—put this where normal chat inputs would go.

{ "model": "gpt-4", "messages": [ { "role": "user", "content": "{{question}}" } ] }
E

Response Content Path: After Trust AI gets the API result, it needs to locate the text answer within potentially dense metadata. Use 'Dot Notation' for the location.

Example for OpenAI: choices[0].message.content
2

Start Auto Evaluation: Click the Start button. Trust AI begins executing background networks immediately, rendering an Expected Time breakdown, while simultaneously showcasing tracked metrics and live latest scores in a real-time monitor panel.
